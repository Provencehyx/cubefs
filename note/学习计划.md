# CubeFS 学习计划

> 适用对象：京东 CubeFS 存储组校招新人
> 已完成：MetaNode 自顶向下 7 层解析（启动 → 网络 → 分发 → Raft → Inode → Dentry → 持久化）
> 学习原则：自顶向下、由点及面、读码 + 画图 + 笔记三结合

---

## 阶段 0：背景与全局观（1～2 天）

**目标**：在动手读新模块前，先把 CubeFS 在分布式存储里的位置、整体角色拎清楚。

- [ ] 通读 [README.md](../README.md)、[docs-zh/](../docs-zh/) 中的总体架构章节
- [ ] 阅读 [ROADMAP.md](../ROADMAP.md)、[CHANGELOG.md](../CHANGELOG.md) 了解版本演进与当前重点方向
- [ ] 画一张**全局架构图**：Client / Master / MetaNode / DataNode / ObjectNode / BlobStore / FlashNode 之间的关系
- [ ] 输出笔记：`note/00-架构总览.md`

**关键问题**：
1. CubeFS 同时支持哪几种访问协议（POSIX / S3 / HDFS / 块）？分别走哪条路径？
2. 副本子系统（DataNode）和纠删码子系统（BlobStore）的定位差异？
3. 元数据、数据、对象网关三类节点是怎么协作完成一次读写的？

## 学习方法约定

1. **每个阶段先画图后读码**：组件交互图 → 类图 → 时序图
2. **每个模块写一份笔记**：放在 `note/` 下，文件名带阶段编号
3. **困惑点先记 TODO，不打断主流程**：每周回头集中攻克
4. **横向对比**：看到新模块时，主动和 MetaNode / DataNode 对照看相同的问题（启动、Raft、持久化、心跳）
5. **遇到名词先去 [proto/](../proto/) 找定义**：CubeFS 的协议结构体集中在这里
